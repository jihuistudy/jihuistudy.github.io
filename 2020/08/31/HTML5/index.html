2.如何区别HTML与HTML5？

-概念：HTML即超文本标记语言，标准通用标记语言下的一个应用。“超文本”就是指页面内可以包含图片、链接，甚至音乐、程序等非文字元素。HTML5是HTML的第五次重大修改而成的，可以理解为升级版，但里面的内容是非常丰富的　　
-区分 html 和 html5有很多方法，下面举例看一下区别：　　
1.在文档类型声明上　　
html:
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN">
HTML5:
<!DOCTYPE html>　
说明：在文档声明上，html代码很长，而html5只有简简单单的声明。　　
2.在结构语义上　　
html4.0：没有体现结构语义化的标签　　
html5：在语义上却有很大的优势。提供了一些新的html5标签，比如:
<header> 、<nav>、<article>、<aside>、<footer>

3.为什么HTML5只需要写<!DOCTYPE HTML>?

HTML5不基于SGML(标准通用标记语言)，因此不需要DTD(文档类型定义)，但是需要doctype来规范浏览器的行为。而HTML基于SGML，需要DTD，才能告知浏览器所使用的文档类型。

7.cookie的缺点

-cookie的数量和长度有限制
-cookie不安全，如果被别人拦截了，就可以取得所有的session信息
-有些数据保存在客户端没有用

8.描述一下sessionStorage和localStorage，localStorage和cookie，cookie和session的区别

sessionStorage和localStorage区别

-sessionStorage是会话存储，存储内容只在同一个会话页面才能访问，当会话结束后，数据也会随之销毁。
-localStorage是本地离线存储，除非主动删除数据，否则存储的数据是永远不会过期的。

localStorage和cookie区别

-localStorage是本地离线存储，概念与cookie类似，localStorage是为了更大容量的存储设计的，有一系列的setItem、getItem等方法操作存储数据，不用自己封装。
-cookie大小受限，每一次请求一个新页面时cookie会被发送过去，无形中浪费了带宽，存储数据的操作要自己封装。但是cookie也是不可或缺的，因为cookie的作用是与服务器进行交互，并且还是HTTP规范的一部分。

cookie和session区别

-cookie数据存放在客户端的浏览器上，session数据存放在服务器上
-cookie不安全，如果被别人拦截了，就可以取得所有的session信息。session相对安全，要将登录信息等重要信息存放在session中。
-session数据保存在服务器上，当访问增多时，会占用较多服务器的资源，cookie是为了减轻服务器的负担。
-cookie的数量和大小有限制，大小不超过4KB，数量不超过20个

9.canvas和SVG的区别是什么？

-svg绘制出来的每一个图形的元素都是独立的DOM节点，能够方便的绑定事件或用来修改。canvas输出的是一整幅画布，不能绑定事件或修改。
-svg输出的图形是矢量图形，后期可以修改参数来自由放大缩小，不会失真和锯齿，与分辨率无关。而canvas输出的是位图，就像一张图片一样，放大会失真或者锯齿，与分辨率有关。
