圣杯布局与双飞翼布局究极解析

发表于 2020-09-03 分类于 前端
本文字数： 6.7k 阅读时长 ≈ 6 分钟

圣杯布局和双飞翼布局两者的功能相同，都是为了实现一个两侧宽度固定，中间宽度自适应的三栏布局。

圣杯布局

实现思路：

1.在html中,先定义好 header和 footer的样式,使之横向撑满。
2.在container中的三列设为浮动(float:left)
3.给外层的父容器container设置padding:0 150px 0 200px;空出左右区域的位置，同时添加overflow:hidden;语句形成BFC，解决父容器高度塌陷的问题
4.center要放在最前面,中间部分在DOM结构上优先，以便先行渲染
5.三列的左右两列分别定宽200px和150px,中间部分center设置100%撑满父元素container,左右两块区域被挤到下一行
6.接下来设置left区域的左边界margin-left:-100%;,让left区域回到上一行父容器的最左侧
7.但这会把center区域遮住一部分,所以通过对左区域left设置相对定位，1eft:-200px;把left拉到父元素外面，定位到页面的最左侧，填补父元素空出的左内边距
8.同样的,对于right区域,设置 margin-left:-150px;把right拉回第一行，使用相对定位，设置right:-150px;填补父容器空出的右内边距

圣杯布局的优缺点

  • 优点：不需要添加多余的DOM节点
  • 缺点：关于页面无限变窄，布局会错乱

关于页面无限变窄，布局错乱问题的解决方法

要设置页面body的最小宽度min-width，最小宽度的值=左区域固定宽度*2+右区域固定宽度
因为当中间自适应宽度小于左区域宽度，当左区域设置负边距和中间区域挤入同一行时，父容器的宽度没有左区域宽度宽，左区域永远也不会和中间区域在同一行,而右区域和左区域相邻，浮动元素会贴着相邻的浮动元素的边界布局，所以右区域与左区域都会显示在下一行

双飞翼布局

实现思路：

为了让中间区域的内容不被遮挡，直接在中间区域外包裹一个父容器，添加一个DOM节点。中间区域用于放置（文本）内容，在存放内容的中间区域用margin-left和margin-right为左右两栏留出位置，左右两栏布局的实现方式和圣杯布局类似都是设置负边距实现，不同的是不需要再使用position定位。

双飞翼布局的优缺点

  • 优点：不用设置页面最小宽度，布局不会变形，不需要使用position定位
  • 缺点：多加了一层DOM节点

其他两端宽度固定，中间自适应的三栏布局的实现方法

方法1：弹性布局

解析思路：

给父容器一个display:flex，left和right给固定的宽度，center区域flex:1自适应，给left区域设置order:-1，让他它排在最左边。

页面无限变窄也不会出现布局错乱的问题

方法2：利用怪异盒子模型+float布局

怪异盒子模型：border-box — 宽高包括了内边距和边框

解析思路：

给中间区域的盒模型设置为border-box，并且设置左区域与右区域宽度相同的左右内边距，用来存放左右区域内容，以防遮挡中间区域内容。使用浮动和负边距使左右区域放入中间区域的左右内边距位置。

  • 为什么要把中间区域的盒子模型设置为怪异盒子模型？
    因为中间区域的宽度为100%，若为标准盒模型，设置内边距时，会在原来100%的宽度上在增加元素宽度，使得元素的宽度超过页面的宽度，使页面出现横向滚动条。而包裹中间区域的父元素container的宽度依然是页面宽度，左区域left的左边界的%是根据父元素的宽度来参照的，所以左区域不能到达中间区域的左边距的位置，故不能达到效果。
    而使用IE/怪异盒子模型，添加内边距，会使元素两边的空白区域变大，中间文字区域往里缩，中间区域的宽度依然为100%（即页面宽度），故使用浮动和负边距可以达到效果。
